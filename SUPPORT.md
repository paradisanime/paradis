## Before Opening an Issue

```
  1. Read docs/INSTALL.md and docs/FAQ.md
  2. Confirm you installed from this repository's Releases
  3. Note your APK version / versionCode (Settings or About if available)
  4. Capture Android version + device model
```

## How to Get Help

```
  Product bugs / install issues
  ............................. Open a GitHub Issue using the templates

  Security reports
  ................ Follow SECURITY.md (prefer private disclosure)

  Feature ideas
  ............. Use the feature-request template; keep scope realistic
```

## What Helps Maintainers Most

```
  . Exact steps to reproduce
  . Screenshots / short screen recordings when UI is involved
  . Whether the issue happens on Wi-Fi, mobile data, or both
  . Whether you are signed into AniList
  . Whether it happens on one title or all titles
```

## Response Expectations

```
  This is a public release / documentation repository.
  Response timing depends on maintainer availability.
  Issues without reproduction details may be closed as incomplete.
```
